2.6. IAMのフォーマット¶

公式ドキュメントのIAMのCloudFormationのテンプレートリファレンス

2.6.1. Policy¶

基本的には、PolicyDocumentに作成したPolicyを記述する。

Policy:
    Type: "AWS::IAM::ManagedPolicy"
    Properties:
        Path: "/"
        ManagedPolicyName: !Sub "policy-[xxx]"
        Description: "[Description Your Policy]"
        PolicyDocument:
            Version: "2012-10-17"
            Statement:
                - 
                    Effect: Allow
                    Action:
                        - "[Target Action]"
                    Resource: 
                        - "[Target Resource]"

2.6.2. Role¶

AssumeRolePolicyDocumentに信頼ポリシーの記述を、ManagedPolicyArnsにRoleを付与するPolicyの参照を記述する。 Policiesというセクションでインラインポリシーを作成することもできる。

Role:
    Type: AWS::IAM::Role
    Properties:
        Path: "/"
        RoleName: !Sub "role-xxx"
        Description: "[Description Your Role]"
        MaxSessionDuration: 3600
        AssumeRolePolicyDocument:
            Version: 2012-10-17
            Statement:
                - 
                    Effect: Allow
                    Principal:
                        Service:
                            - [principal].amazonaws.com
                    Action:
                        - sts:AssumeRole
        ManagedPolicyArns: 
            - [Policy For This Role]
        Tags:
            - 
                Key: CreatedBy
                Value: !Ref "AWS::StackName"
